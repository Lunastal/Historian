# Historian
Historian is a script designed for AI Dungeon scenarios.

## Description
AI Dungeon scenarios manage how history and memory are passed to the underlying LLM for generating subsequent story lines. However, this history and memory are constrained by token limits and the physical amount of data that can be fed to the model. As a result, the story—and the characters within it—will gradually begin to forget key plot developments after enough actions have been taken.

Historian aims to mitigate this "memory loss" by inserting relevant information into appropriate Story Cards, so that it is naturally picked up by AI Dungeon's native system, as if the players had created it themselves. The information Historian provides is also designed to be token-efficient and precise enough to effectively "remind" the AI of important story developments.

## Mechanism
Historian works by injecting additional prompts into the context, prompting the underlying LLM to produce extra plot-related snapshots alongside the normal story narrative.

The additional information generated by these prompts intentionally avoids character emotions and personalities, since those aspects are primarily supplied by the AI itself. With dynamic options that rotate between different AI models, an emotional state produced by one model (e.g., an angry tone) would be irrelevant—or even counterproductive—when another model takes over (e.g., a calmer one).

Instead, Historian records concrete details that are easily lost due to the limitations of the history and memory system, helping the story and character actions stay on track.

## Types of Historian Story Cards

### Settings
- Name: ⚙️ Settings
- Purpose: Settings, on/off switches used by Historian.

### Master Journal
- Name: Master Journal
- Purpose: A Story Card with no trigger, mainly for keeping track of recorded events.
- Content: 4W sentences generated by the AI.
- Archiving: The main entry has a character limit (default: 1775). Old sentences will be moved to the Story Card notes.

### Sub Journal

1. The Sub Journal
   - Name: Journal - {Action Count}
   - Purpose: A Story Card with the player's name as trigger, for keeping the story on track.
   - Content: A 4W (When, Where, Who, What) sentence for a recent promise related to the player.
   - Limit: Allows only 1 sub journal at any time by default.
2. Who Card
   - Name: Journal - {Action Count} - Who
   - Purpose: A Story Card that supplements the "Who" part of the sub journal, describing the person mentioned in the sentence.
   - Content: A short description of the character.
3. Where Card
   - Name: Journal - {Action Count} - Where
   - Purpose: A Story Card that supplements the "Where" part of the sub journal, describing the place mentioned in the sentence.
   - Content: A short description of the place.
  
### Character Activity Log
   - Name: {Character Name}'s Log
   - Purpose: A Story Card with the character's name as trigger, for remembering what happened to the character.
   - Content: A third-person description of a recent event.
   - Limit: Allows only 1 activity log for each character at any time by default.
   - Archiving: The main entry has a character limit (default: 775). Old sentences will be moved to the Story Card notes.
  
### Character Bond
   - Name: {Character Name}'s Bond
   - Purpose: A Story Card with the character's name as trigger, for describing the current relationship between the player and the character.
   - Content: A third-person description of the relationship.
   - Limit: Allows only 1 bond log for each character at any time by default.
   - Archiving: There is no archiving for bond cards. Every time the content is updated, it is completely replaced by the newer entry.

### Additional Information
A sub journal has an expiry (default: 10 actions). Once expired, the sub journal card, who card, and where card will be removed automatically.

_Except for the Who and Where cards, all contents are prefixed with "{Action Count}: ", which doubles as the last triggered history for calculating delays._

## Usage & Settings

**Settings**
- Enable: true/false. Set to 'true' to use the script.
- Elaborate Who: true/false. An additional card will be generated for the person mentioned in the journal.
- Elaborate Where: true/false. An additional card will be generated for the place mentioned in the journal.
- Characters: A list of main characters (excluding yourself), separated by commas.
- Record Activity: true/false. Records the activities of a character.
- Record Bond: true/false. Records the latest relationship between you and a character.
- Auto Add Character: true/false. Adds newly identified characters from journals into the character list, if fewer than 3 characters are registered.

**Supported AI models**
- DeepSeek, DeepSeek V4 Flash, Equinox, Fable, Gemma 4 31B, Harbinger, Hearthfire, Madness, Muse, Nova, Wayfarer Large, Wayfarer Small 2

**Hidden settings (If not pulled into the Settings Card)**
- The script won't start before the 8th action.
- Only 1 active journal at a time (Do things one by one!).
- Journals expire after 10 actions.
- Character log has a delay of 8 actions, but journal takes priority.
    > Base delay = 8 actions. Increases based on the number of characters listed, starting from the 4th person onward.
- The AI decides which character's log to update based on the number of mentions in the recent story.

**Empty output**
- In case of an empty output, erase and continue might work...

## Install into a scenario

1. On the Details tab in the scenario edit page, look for "Scripting" and enable "Scripts Enabled".
2. Click the Edit Scripts button.
3. Copy the contents of the file _library.js_ and paste them into "Library".
4. Copy the contents of the file _input.js_ and paste them into "Input".
5. Copy the contents of the file _context.js_ and paste them into "Context".
6. Copy the contents of the file _output.js_ and paste them into "Output".
7. Save and test!
